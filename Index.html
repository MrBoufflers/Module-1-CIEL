a. Les bases de l'informatique (historique)

L'informatique n'est pas née avec les smartphones. C'est une longue histoire d'idées et d'inventions.

  • Les débuts : On peut remonter aux machines à calculer mécaniques (Blaise Pascal au 17ème siècle).
  • Le concept : Au 19ème siècle, Charles Babbage imagine une "machine analytique", le premier concept d'ordinateur programmable, avec l'aide d'Ada Lovelace, considérée comme la première programmeuse.
  • L'ère électronique : Les premiers véritables ordinateurs électroniques apparaissent pendant la Seconde Guerre mondiale, comme l'ENIAC. C'étaient des monstres occupant des pièces entières.
  • La révolution du microprocesseur : Dans les années 1970, l'invention du microprocesseur permet de réduire la taille et le coût des ordinateurs. C'est le début de l'informatique personnelle avec des entreprises comme Apple et Microsoft.
  • Aujourd'hui : L'informatique est partout : dans nos poches (smartphones), nos voitures, nos maisons.

b. Comment marche un PC théoriquement ?

Presque tous les ordinateurs modernes fonctionnent sur un principe simple appelé l'architecture de Von Neumann. Elle repose sur quatre éléments principaux qui travaillent ensemble.

Périphériques E/S
<=>
CPU
↓↑
Mémoire Vive (RAM)
Survolez un composant pour voir sa description.

Le processus est simple : quand vous lancez un programme, le CPU va chercher ses instructions sur le disque de stockage, les charge dans la RAM pour y accéder rapidement, et exécute les calculs. Le résultat est ensuite affiché sur un périphérique de sortie, comme l'écran.

c. Les composants d'un ordinateur et leurs rôles

Processeur (CPU): Le cerveau. Exécute les calculs et les instructions.
Carte Mère: Le système nerveux. Connecte tous les composants.
Mémoire Vive (RAM): Mémoire à court terme. Stocke les données en cours d'utilisation.
Stockage (SSD/HDD): Mémoire à long terme. Garde vos fichiers et l'OS.
Carte Graphique (GPU): Spécialisée dans l'affichage et les calculs graphiques.
Alimentation (PSU): Le cœur. Fournit l'énergie à tout le système.
Boîtier (Case): La carrosserie qui protège tous les composants.

d. Différence hardware, firmware et software

Hardware (Matériel): Tout ce qui est physique et tangible. Le CPU, l'écran, la souris... C'est la machine elle-même.
Software (Logiciel): L'ensemble des programmes et instructions qui disent au hardware quoi faire. C'est immatériel. Ex: Windows, un jeu, un navigateur.
Firmware (Micrologiciel): Un logiciel spécial, "gravé" dans un composant matériel pour le faire fonctionner à un bas niveau. Le BIOS de la carte mère est l'exemple parfait.

Module 2: Introduction à Windows

a. Qu’est-ce qu’un système d’exploitation ?

Un OS (Operating System), c'est le chef d'orchestre de l'ordinateur. C'est le logiciel principal qui fait le lien entre le matériel (hardware) et vous (l'utilisateur) avec vos logiciels (software). Sans OS, un ordinateur n'est qu'une boîte de métal et de plastique inutile.

Ses rôles principaux :

  • Gérer le matériel : Il parle au CPU, à la RAM, au disque dur... pour que tout fonctionne ensemble.
  • Gérer les fichiers : Il organise les données sur le disque dur avec des dossiers et des fichiers.
  • Fournir une interface utilisateur : C'est ce que vous voyez à l'écran : le bureau, les icônes, les fenêtres (GUI - Graphical User Interface).
  • Exécuter les logiciels : Il donne aux applications (Chrome, Word...) les ressources dont elles ont besoin pour fonctionner.

Petit historique : Au début, tout se faisait en ligne de commande (comme MS-DOS). Puis sont arrivés les OS avec des fenêtres et une souris, rendant l'informatique accessible à tous.

b. Les principaux OS : Windows, Linux et macOS

Windows

Le plus répandu. Grande compatibilité logicielle. Facile d'accès pour les débutants.

macOS

Exclusif aux appareils Apple. Réputé pour sa stabilité et son design soigné.

Linux

Open source, gratuit et ultra-personnalisable. Le roi des serveurs.

c. Qu’est-ce que le BIOS ?

Le BIOS (Basic Input/Output System) ou son successeur moderne l'UEFI, est le tout premier programme qui se lance quand vous appuyez sur le bouton d'alimentation. C'est un firmware sur la carte mère.

À quoi ça sert ?

  1. Vérifier le matériel (POST) : Un rapide check-up pour s'assurer que tout fonctionne.
  2. Configurer le matériel : Permet des réglages de bas niveau (ex: ordre de démarrage).
  3. Lancer l'OS : Sa mission finale est de trouver Windows sur un disque et de lui passer la main.

Module 3: Introduction au Développement Software et Web

a. Application lourde vs Site web

Application lourde (Logiciel)

Installée sur votre PC (ex: Word, Photoshop). Elle est souvent plus performante et peut fonctionner hors-ligne, mais nécessite une installation et des mises à jour manuelles.

Application Web (Site)

Accessible via un navigateur (ex: Gmail, Netflix). Pas d'installation, accessible partout, mises à jour transparentes, mais requiert une connexion internet.

b. Le web : Les protocoles de communication

Pour que les ordinateurs puissent communiquer sur un réseau, ils doivent parler le même langage et suivre les mêmes règles. C'est ce qu'on appelle un protocole.

  • TCP/IP : La base d'Internet. C'est un ensemble de protocoles qui assure que les données envoyées d'un point A arrivent à un point B, sans erreur.
  • HTTP/HTTPS : Le protocole du Web. Il permet à votre navigateur de demander des pages web à un serveur et de les recevoir. Le 'S' de HTTPS signifie "Sécurisé" (les données sont chiffrées).
  • FTP : Le protocole pour transférer des fichiers.
  • SMTP, POP, IMAP : Les protocoles pour les emails.

c. Le protocole HTTP et les API en détail

Une requête HTTP est comme une commande au restaurant. Elle contient des "verbes" (méthodes) qui indiquent ce que vous voulez faire :

  • GET : "Donnez-moi la page d'accueil." (Récupérer des données)
  • POST : "Voici mes informations pour créer un compte." (Envoyer de nouvelles données)
  • PUT / PATCH : "Je veux modifier mon profil." (Mettre à jour des données existantes)
  • DELETE : "Je veux supprimer mon commentaire." (Supprimer des données)

Une API (Application Programming Interface) est une "carte" qui expose ces actions au monde extérieur. Elle définit des URLs spécifiques (des "endpoints") où d'autres applications peuvent envoyer des requêtes GET, POST, etc., pour interagir avec les données du serveur de manière contrôlée.

Module 4: Introduction au Codage

Références au Référentiel CIEL

Compétence visée : C08 - Coder.

Savoirs associés : S5.1 (Langages de développement), S5.5 (Principes fondamentaux de programmation).

a. Définition et historique

Coder (ou programmer), c'est donner une série d'ordres précis à un ordinateur dans un langage qu'il peut comprendre. L'ordinateur est un exécutant parfait mais sans initiative : il ne fait que ce qu'on lui dit. Le rôle du développeur est de traduire un besoin en une suite d'instructions logiques et simples.

b. Les briques de base de la programmation

Les Variables

Une variable, c'est comme une boîte avec une étiquette. On peut mettre quelque chose dedans (un nombre, du texte...) et on y fait référence avec son étiquette.

age = 17
nom = "Dupont"
print(nom, "a", age, "ans.")

Les Conditions (`if`, `elif`, `else`)

Permet à un programme de prendre des décisions. "S'il pleut, prends un parapluie, sinon, prends des lunettes de soleil".

# Cas d'usage : Gestion de ticket
statut_ticket = "Nouveau"
if statut_ticket == "Nouveau":
    print("Ticket assigné à un technicien.")
elif statut_ticket == "En cours":
    print("En attente de la réponse du technicien.")
else:
    print("Le ticket est fermé.")

Les Boucles `while` (Tant que)

Répète un bloc d'instructions TANT QU'une condition est vraie.

# Cas d'usage : Traiter tous les nouveaux tickets
tickets_nouveaux = 3
while tickets_nouveaux > 0:
    print("Je traite un nouveau ticket.")
    tickets_nouveaux = tickets_nouveaux - 1
print("Plus de nouveaux tickets à traiter.")

Les Boucles `for` (Pour)

Répète un bloc d'instructions un nombre de fois défini, parfait pour parcourir une liste d'éléments.

# Cas d'usage : Envoyer un rappel pour chaque ticket assigné
tickets_assignes = ["Ticket001", "Ticket005", "Ticket012"]
for ticket in tickets_assignes:
    print("Email de rappel envoyé pour le", ticket)

Étape 5 : Le jeu complet "Plus ou Moins"

Maintenant, assemblez tous ces concepts pour créer le jeu final.

import random
import time

while True:
    print("Je pense à un nombre entre 1 et 100...")
    nombre_secret = random.randint(1, 100)
    
    while True:
        proposition_str = input("Devine le nombre : ")
        
        if not proposition_str.isdigit():
            print("Erreur : Veuillez entrer un nombre.")
            continue
            
        proposition = int(proposition_str)
        
        if proposition < nombre_secret:
            print("C'est plus grand !")
        elif proposition > nombre_secret:
            print("C'est plus petit !")
        else:
            print("Bravo ! C'était bien", nombre_secret)
            time.sleep(5)
            break
            
    rejouer = input("Veux-tu rejouer ? (oui/non) ")
    if rejouer.lower() != "oui":
        break

Module 5: Les Bases de Données

Références au Référentiel CIEL

Compétence visée : C04 - Analyser une structure matérielle et logicielle.

Savoirs associés : S3.1 (Infrastructures), implicitement lié à C08 (Coder).

a. Définition et concepts clés

Une base de données (BDD), c'est une manière organisée et structurée de stocker de l'information pour pouvoir la retrouver et la manipuler facilement. On s'intéresse au type le plus courant : les bases de données relationnelles.

L'idée est de ranger les données dans des tables (comme des tableaux Excel).

  • Table : Représente un type d'objet (ex: `Eleves`).
  • Champ (ou Colonne) : Une caractéristique de cet objet (ex: `nom`).
  • Enregistrement (ou Ligne) : Un objet spécifique (ex: l'élève Dupont).

Les Clés : le Super-Pouvoir des BDD Relationnelles

  • Clé Primaire (PK) : Un identifiant unique pour chaque ligne d'une table (ex: `id_eleve`). Elle ne peut pas être vide.
  • Clé Étrangère (FK) : Un champ dans une table qui fait référence à la clé primaire d'une autre table. C'est ce qui crée la relation.

b. Modélisation : exemple progressif d'un blog

Étape 1 : Les articles. On commence avec une simple table pour stocker nos articles.

Table `Articles`

id_article (PK)titrecontenu
1Mon premier article...
2Un autre sujet...

Étape 2 : Ajout des auteurs (Relation 1-N). Un auteur peut écrire plusieurs articles, mais un article n'est écrit que par un seul auteur. C'est une relation "un à plusieurs". On ajoute une table `Auteurs` et une clé étrangère dans `Articles`.

Table `Auteurs`

id_auteur (PK)nom_auteur
10Alice
11Bob

Table `Articles`

id_article (PK)titreid_auteur (FK)
1Mon premier article10
2Un autre sujet10

Étape 3 : Ajout des catégories (Relation N-N). Un article peut appartenir à plusieurs catégories (ex: "Tech" et "Web"), et une catégorie peut contenir plusieurs articles. C'est une relation "plusieurs à plusieurs". On a besoin d'une table de liaison.

Table `Articles`
  • id_article (PK)
  • titre
Table de liaison `Article_Categories`
  • id_article (FK)
  • id_categorie (FK)
Table `Categories`
  • id_categorie (PK)
  • nom_categorie
